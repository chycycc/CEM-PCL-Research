# EPCL v6.4 Ablation Study Plan (消融实验规划)

## 1. 实验目标
通过控制变量法，严格验证 EPCL v6.4 架构中三个核心机制（EPCL、Dropout、Early Stop）对分类准确率和生成共情度的独立贡献。

## 2. 核心消融设置

为了实现学术严谨性，我们需要训练以下三个变体模型，并与 Baseline(v4/v5) 及 v6.4 (Full) 处于同一实验环境中对比：

| 变体名称 | 对应缺失机制 | 预期影响（理论推演验证） |
| --- | --- | --- |
| **w/o EPCL** | 移除拓扑锚定，退化为纯分类 | 细粒度情感边界重合，主客体混淆增加，分类准确率下降 |
| **w/o Dropout** | 移除 Decoder 侧或特征层 Dropout | 分类头严重过拟合，导致部分类别（如 impressed/embarrassed）发生极性反转 |
| **w/o Early Stop** | 移除 14k 步的分类梯度截断 | 分类准确率可能持平，但 PPL 显著恶化，重现“跷跷板效应” |

## 3. 执行步骤
1. **构建变体代码**：在 `main.py` 和 `model_epcl.py` 中添加控制开关（如 `--ablation_mode`）。
2. **重训模型**：使用一致的随机种子和超参数，分别跑完 20k steps。
3. **指标提取**：提取各变体的最佳 PPL、最佳 Acc，以及在最佳 PPL 处的对应 Acc（双超矩阵）。
4. **Case 对齐**：针对 `v64_final_archive` 中整理出的 3 个经典 Case，提取这三个变体模型的预测与生成结果，填入诊断报告中的消融对比表。

## 4. 注意事项
- 所有变体的显存占用和训练时间需记录，以排除计算资源的偶然性影响。
- 确保测试集的预处理与评估脚本 (`evaluate.py`) 完全一致。
